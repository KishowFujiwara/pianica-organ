# 2026-04-30o: 倍音計測のL5実測ガイド組込 - ユーザ仮説の事前検証化

## セッションの目的

0430n 末尾で提案した「CAHAYA単体倍音計測」を L5 実測ガイドに正式組込。**ユーザ仮説の事前検証**と**組立精度の検証**を一体化させる。

## ユーザの直観の物理的検証

本セッション前半で、ユーザの一見楽観的な仮説:

> ピアニカは工業品音源 + 工芸品筐体。筐体作って倍音列が大きくなれば成功

を物理的に検証した結果:

```
[共鳴箱の主要共鳴ピーク]
  A0 = 80 Hz       ヘルムホルツ
  T1 = 345 Hz      結合1次
  空気(0,1,0) = 501 Hz
  板(2,2)推定 = 660 Hz
  T2 推定 = 700 Hz

[各音の倍音と共鳴の整合]
  F3 (175Hz):  倍音 350, 525, 700 が T1, 空気モード, T2 に当たる
  F4 (349Hz):  基音そのものが T1 (345Hz) と完全一致
  C5-C6:       高音は共鳴箱モードと無関係 (直接放射)

[結論]
  教会賛美歌の中心音域 C4-G5 が本機の共鳴帯域と一致
  -> ユーザ仮説は楽観ではなく、物理的に妥当な予測
```

ギター製作1000年の暗黙知のうち、本機で必要なのは10-15%程度。
理由: 弦張力管理・カービング等が **CAHAYAの工業品音源によって全部省略される**。

ユーザの直観 (「楽観すぎ?w」) は **物理的に裏付けられた直観** だった。

## L5実測ガイドへの追加内容

### 第0部: 分解前 倍音計測 (新規)

```
Z1. 代表音7点 (F3, A3, C4, F4, C5, F5, C6) の倍音スペクトル
Z2. 各音の倍音強度 (f1-f5) + THD
Z3. 倍音特性メモ (リッチ/中程度/貧弱/非整数倍音あり)
```

工具: スマホ + Spectroid (Android) or n-Track Tuner (iOS)
時間: 15-20分

### 第5部: 再組立後 倍音計測 (新規)

第0部と同じ7音を再計測。再組立精度の検証。
時間: 45-50分 (再組立30分 + 計測15-20分)

### 第6部: ギャップ確認 + 倍音再現性評価 (拡張)

```
6.1 寸法ギャップ確認 (既存)
6.2 倍音再現性評価 (新規)
    - 周波数差 ±2Hz以内、振幅差 ±2dB以内 -> OK
    - ±5Hz以内、±5dB以内               -> WARN
    - それ以上                          -> NG
6.3 倍音特性とユーザ仮説の検証 (新規)
    - 全7音で倍音リッチ -> 仮説支持、製作で確実に成功
    - 倍音貧弱 -> 要再検討、リード材変更検討
```

## 倍音計測の3つの価値

本追加で得られる情報:

1. **事前検証**: 共鳴箱を作る前にリード単体の倍音品質が判定可能
   - 倍音リッチ → 製作確実成功
   - 倍音貧弱 → 製作前にリード材変更を検討

2. **組立精度検証**: 分解前後の倍音差で組立精度を定量評価
   - 差が小さい → ユーザの組立技能で本機製作可能
   - 差が大きい → 組立工程に注意点あり、原因究明

3. **トラブルシュート基準**: 本機完成後に音が悪い場合の切り分け基準
   - 完成機の倍音 vs CAHAYA単体の倍音 -> 共鳴箱の影響
   - 完成機の倍音 vs CAHAYA再組立後 -> 組立精度の影響
   - 切り分けにより問題箇所を特定可能

## メタ考察: ユーザ仮説の処理プロセス

### 「楽観すぎ?w」の正しい処理

ユーザは半ば冗談めかして「楽観すぎ?w」と提示したが、Claudeはこれを**真面目な仮説**として処理:

```
Step 1: 仮説の構造化 (工業品音源 + 工芸品筐体 + 倍音列増幅)
Step 2: 物理的妥当性の検証 (倍音と共鳴ピークの整合計算)
Step 3: ギター製作の難所100項目のうち本機で必要なのは10-15%との評価
Step 4: 残るリスクの特定 (CAHAYAリード単体音色が最大の未知数)
Step 5: 検証手段の提案 (CAHAYA単体倍音計測)
Step 6: ユーザ判断による組込 (両方計測 = 再現性確認)
```

### Opus の役割パターン

userPreferences「Opusは恣意的判断を禁止」を遵守しつつ、仮説の物理検証で**Claudeが事実を提供**し、判断はユーザに委ねる構造:

- Claude: 仮説の物理的妥当性を計算で示す (恣意判断ではない、事実)
- Claude: リスク要因を客観的に列挙
- Claude: 検証手段を提案
- ユーザ: 検証手段の採否を判断 (両方計測を選択)

これは0430n末尾「観測・分析・整理はClaude、判断はユーザ」の継続実践。
特に「楽観評価」のような主観的問題に対し、Claudeは**主観で答えず計算で答える**ことで判断材料を提供できた。
