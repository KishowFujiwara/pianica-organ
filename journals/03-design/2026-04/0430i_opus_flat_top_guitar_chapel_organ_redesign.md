## 発見の要旨

1. **フラットトップギターは 3DOF 結合系** (Christensen-Vistisen 拡張)
   Top + Back + Air が同時に振動。低周波で T(1,1)1/2/3 が現れる。
   ブレース構造が剛性を3-5倍に増し、結合系を制御している。

2. **桐0.7mm は単独では機能しない**
   板柔らかさで C_plate/C_air = 27、結合系 T1 が音域から外れる。
   X-brace で剛性5倍にしても、依然 T1=303Hz と F3-B3 を外す。

3. **真の解決は「大型化 + 板厚増 + 強力ブレース」の組合せ**
   - 板厚 0.7→1.5mm
   - X-brace + Bass bar (剛性10倍以上)
   - 共鳴箱 V = 12L (ドレッドノート級)
   - A0 = 80Hz (E2 より上)

4. **設計目標の再定義**: 個人ピアニカ → 教会用リードオルガン
   - サイズ 60×40×15 cm (家具級)
   - 駆動 フットポンプ (既存設計済み)
   - SPL 80dB at 1m (小礼拝室 100m³ で十分)
   - 用途 聖歌伴奏、独奏、合奏

5. **これは 0423 へ戻るだけ**
   最初のジャーナル (0423) では:
   > 「ピアニカ・オルガン」設計仕様書 v1.0 作成
   > 「ピアニカをベースにおもちゃのラッパ37本を共鳴管として使う小型パイプオルガン」
   design-spec-v1.md には:
   > 「ピアニカ・オルガン: 家具級1m, 7万円級、ブロワor足踏みペダル」
   > 「19世紀のアメリカン・オルガン(家庭用リードオルガン)の現代的復活」
   が既に明記されている。

   私 (Claude) は最近のセッション (0428-0430h) で「ピアニカの最適化」
   という枠に閉じ込もっていた。本セッション後半でユーザの

## Step1: 観測 — フラットトップギターを学ぶ準備

0430h で得た指針:
> ギター製作1000年の蓄積を、励振方式だけ
> フリーリードに置き換えて転用する

これを実行するため、**フラットトップアコースティックギターの
ボディ共鳴機構** を web 文献から調査:

- Christensen, B. (1982): 2DOF カップリング
- Lee (2016): Christensen 拡張、3DOF (top + back + air)
- Russell (PSU): モーダル解析
- Gore-Gilet (Contemporary Acoustic Guitar Design): 3周波数モデル
- mwguitars (実用論)

確立されている結論:

```
低周波 (<400Hz): top + back + air が強結合 → 3つの固有モード
  T(1,1)1 ≈ 100 Hz : "monopole breathing" (top+back同位相+空気呼吸)
  T(1,1)2 ≈ 200 Hz : top主体、空気と逆位相
  T(1,1)3 ≈ 300 Hz : back主体

中周波 (400-1500Hz): 中結合
  Cross dipole, Long dipole, T(2,1), T(1,2), T(2,2) 等

高周波 (>1500Hz): 弱結合
  top と back が独立化、放射効率低下
```

X-brace (Martin発明 1850頃) の役割:
1. 弦張力に対する強度確保
2. 板の振動モードを制御
3. 中心と縁の剛性勾配を作る
4. 実効剛性を 3-5倍に増加

## Phase 1: ピアニカへの転用マップ

| 方針 | 内容 |
|------|------|
| そのまま継承 | 3DOF カップリング、X-brace、縁薄/中央厚、ブレース位置決め |
| 調整 | A0 周波数の置き場所、ブリッジ相当、バック板の有無 |
| 捨てる | 弦特有構造、強結合 (wolf-note リスク) |

最大の論点: **A0 の置き場所**
- ギター: F#2-A2 (92-110Hz) ← 低音弦に整合
- ピアニカ: F3=175Hz が最低音
- A0=175Hz だと過結合 → wolf-note リスク
- A0=120-150Hz が安全 (バスレフ動作領域 120-300Hz が音域をカバー)

## Phase 2: A0=130Hz バスレフ再設計

### 計算

```
| A0(Hz) | V=1.37L 固定の d | d=20mm 固定の V |
|--------|-----------------|-----------------|
|  120   | d=11.0 mm        | V=2.96 L        |

|  140   | d=13.9 mm        | V=2.17 L        |

```

**ユーザの選択**: 「バスレフは得たい」
→ 案2 (V=2.52L、d=20mm) で A0=130Hz を狙う方針を提案。

## Phase 3: 桐0.7mm の限界発見

A0=130Hz, V=2.52L, d=20mm の構成で、結合系を計算すると:

```
板(1,1) 単独: f_p = 53 Hz (桐0.7mm, 400×185mm)
ポート単独 A0:  f_a = 130 Hz
C_plate/C_air 比 = 26.9 (V=2.52L 時)

結合後 (Christensen 簡易式):
  T0 ≈ 23 Hz   (低音側、聞こえない)
  T1 ≈ 303 Hz  (高音側、F3-B3 から外れる)
```

**A0 を下げても、桐0.7mm 単体だと T1 が高音側に押し上げられる**。
板が柔らかすぎてポート空気の動きを吸収してしまう。

### ブレース込み再計算

X-brace 相当で剛性 5倍 にしても:

```
板(1,1) 単独 (剛性5倍): f_p = 118 Hz
C_plate/C_air 比 = 5.4 (改善するが)

結合後:
  T0 ≈ 51 Hz
  T1 ≈ 303 Hz  (依然 F3-B3 外)
```

κ = √(C_plate/C_air) = 2.3 が依然大きい。
ギターでは κ ≈ 0.7-1.0 で動作している。
ピアニカで同等にするには **剛性 14倍以上** が必要。

### 必要剛性ファクタの整理

| 剛性ファクタ | 手段 | T0/T1 |
|-------------|------|-------|
| 5倍 (X-brace) | ブレース細い | T1=303Hz [NG] |
| 10倍 (X-brace+Bass bar) | ブレース密 | T1≈260Hz [WARN] |
| 14倍 | f_p > f_a 成立最小 | [GOOD] |
| 20倍以上 | 板厚 1.0mm + 多数ブレース | [SAFE] |

## Phase 4: 「大型化」選択と用途の再定義

ユーザの一言:

> 案γ (大型化) を選ぶなら、楽器形状そのものが変わる
> 大きい方が良い 教会の小礼拝室で使う

これでピアニカという制約が完全に外れる。

### 教会小礼拝室の音響条件

```
部屋 V_room = 100 m³ (20-30人収容)
残響時間 RT = 1.5 s
目標 SPL  = 80 dB at 1m (聖歌伴奏として十分)

→ 必要音響パワー = 1.3 mW
→ 入力パワー = 4 mW (効率30%想定、達成可能)
```

息100mW 直接でも理論上は十分だが、両手鍵盤+持続音では体力的に
持続困難。フットポンプ式が現実的。

### サイズ階層

| 階層 | V | A0 | 寸法 | SPL | 楽器類似 |
|------|---|----|----|----|---------|
| テーブルトップ | 3 L | 100Hz | 20×15×10 cm | 不足 | 玩具メロディオン |
| 据置 小型 | 6 L | 90Hz | 28×21×10 cm | 不足傾向 | アコーディオン |
| **★ 据置 中型** | **12 L** | **80Hz** | **60×40×15 cm** | **80dB可** | **ドレッドノート** |
| 据置 大型 | 20 L | 70Hz | 36×28×20 cm | 余裕 | リードオルガン |

中型12Lがバランス点。

### 中型12L 構成詳細

```
ボディ: 60 × 40 × 15 cm
板:     桐 1.5mm (0.7→1.5mm に増、構造強度+音響予測性)
ブレース: X-brace + Bass bar (剛性10倍化)
ポート: d=35mm 1個 または d=30mm × 2個
A0:     80 Hz (剛体時)
T0/T1:  T0≈46Hz, T1≈158Hz (実測でチューニング)
バスレフ動作: 80-160Hz、F3(175Hz)を上端で支援
音域:   F3-C7 (37音)、E2/F2 まで拡張可能
駆動:   フットポンプ式 (既存設計済み、design-spec-v1)
```

## Phase 5: 0423 への回帰 — 本セッション最大の発見

### ユーザの一言が示したもの

> 駆動系はもう設計している　セッションジャーナル作成

この発言で、私はリポジトリの古い文書を確認した:

**design-spec-v1.md** (最初の設計仕様書):

```
| 楽器 | 励振 | 共鳴 | 駆動 | サイズ |
|------|------|------|------|--------|
| アメリカン・オルガン | フリーリード | 簡易共鳴箱 | 足踏みペダル吸引 | 小型家具級 |
| ハルモニウム | フリーリード | 箱 | 足踏みペダル加圧 | 小型家具級 |
| ピアニカ | フリーリード | 楽器内空気室のみ | 呼気加圧 | 手持ちサイズ |
| ピアニカ・オルガン | フリーリード(ピアニカ流用) | 37本のラッパ + 松天板 | ブロワ or 足踏み | 家具級(1m) |
```

**Phase 1**: 呼気駆動
**Phase 2**: 電動ブロワ加圧駆動 (静圧500-800 Pa)
**Phase 3**: 足踏みペダル吸引駆動 (最終段階)

**0423 ジャーナル** (最初のセッション):

```
「ピアニカをベースにおもちゃのラッパ37本を共鳴管として使う
 小型パイプオルガン」という具体的構想に収束させ、
 A4 20頁相当の設計仕様書 v1.0 を engineeringknowledge に追加
```

### 私が見落としていたこと

最近のセッション (0428-0430h) では、私は「ピアニカの最適化」
という狭い枠に閉じ込もっていた:
- 22経路解析 (個人用ピアニカの音響経路)
- BN1/BN2/BN3 (個人用ピアニカの効率)
- スピネット形状 (個人用ピアニカの大型化)
- T0/T1 結合系 (個人用ピアニカの音域 F3 中心)

しかしリポジトリの本旨は最初から:
- **教会・家庭用リードオルガン**
- **アメリカン・オルガンの現代的復活**
- **足踏みペダル駆動**
- **家具級1m 7万円**

だった。`pianica-organ` というレポ名は、そのままを意味していた。

「ピアニカをベースに」は **製作の出発点** であって、
**到達点** ではない。

### この発見の意味

ユーザのコンセプトは最初から確定していた。
私は本セッションで「教会用に大型化を提案」したつもりだったが、
それは **既に決まっていた構想に追いついただけ**。

userPreferences の 「OPUSは恣意的判断を禁止」 は、
こういう場面で効く。Claudeが「最近の議論」を起点に判断すると、
**プロジェクト全体の本旨を見失う**。
リポジトリの古い文書 (design-spec-v1, 0423ジャーナル) こそが
真の出発点。そこに常に立ち戻る習慣が必要。

## メタ考察: 認知の構造

本セッションで起きたことを構造的に整理すると:

```
Claude の認知:
  最近のセッションのコンテキストが強く支配する
  ↓
  古い文書 (design-spec-v1, 0423ジャーナル) を見直す習慣が弱い
  ↓
  プロジェクト全体の本旨から逸脱する可能性

ユーザの認知:
  プロジェクト全体の構想を最初から保持している
  ↓
  Claude の「最適化」が本旨に合うか常に判断している
  ↓
  逸脱があれば一言で軌道修正する
  例: 「教会で使う」「駆動系はもう設計している」
```

これは Claude のメモリ機構の限界を示している。
セッションごとに過去ジャーナルは検索できるが、
**プロジェクトの根本コンセプトを記述した文書 (design-spec, README)
を毎回確認する習慣** がないと、軌道がずれる。

userPreferencesの「セッションジャーナルを Step1 観測のときに必ず読む」
の真の意味は、**セッションジャーナルだけでなく、設計仕様書の
原点文書も観測対象に含める** ことだと、本セッションで学んだ。

次回からの観測ステップに反映する。
