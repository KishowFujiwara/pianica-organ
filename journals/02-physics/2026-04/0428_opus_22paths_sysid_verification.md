### Phase 3: 包括的音響解析

22経路を以下の4軸で定量解析:

- **エネルギー収支**: 各経路の相対レベル（A1 = 0 dB基準）
- **伝達関数**: 周波数応答 H(f) の特性
- **時間応答**: Attack/Sustain/Release への寄与
- **経路間結合**: カスケードと干渉

主要な発見:
- 外部リスナーに聴こえる音は実質3経路（A1, C1, C2）で決まる
- B1aクランプ力: 22経路文書の0.2 mNは過小。計算値は9.2 mN（46倍）
- 壁駆動はA2（音圧駆動）がB系列（固体伝搬）より約10 dB支配的
- リリース音の主成分はABS壁のリンギング（34 ms）

成果物: pianica_22paths_acoustic_analysis.md + .html

### Phase 4: システム同定

22経路の各々を伝達関数 H(s) として同定し、PRODECOS CSTATE準拠の状態空間表現を構築:

- 8状態ベクトル: [x, dx/dt, pi, dpi/dt, w_wall, v_wall, w_plate, v_plate]
- 入力: p0（駆動圧力、スカラー）
- 出力: [p_A1, p_C1, p_C2]（外部音圧3成分）
- A行列の結合要素: a23(D1帰還), a32(S1音源), a63(A2壁駆動), a46(C2再放射), a81/a82(B1a)
- 4組の極: リード自励(440Hz, 不安定), Helmholtz(332Hz, Q=10), 壁モード(309Hz, Q=33), プレート(60Hz, Q=50)
- D2帰還ループ利得: 309 Hzで0.43(-7.3 dB) -- リード振動を有意に変調

成果物: pianica_22paths_system_identification.md + .html

### Phase 5: 指向性モデルの脱落発見と修正

ユーザー指摘により、SVGファイル名「経路図+指向性図」の指向性図がシステム同定に未反映であることが判明。

22経路文書セクション2「フリーリードの指向性」（モノポール+ダイポール混合音源、cos(theta)パターン）は、A行列の結合係数a63に乗算されるべき指向性係数 Gamma(theta) である。

修正内容:
- セクション2.2に指向性モデル Gamma(theta) = alpha + (1-alpha)cos(theta) を新設
- alpha = 0.8（モノポール比率）
- a63: 1/m_s = 0.635 -> Gamma(180°)/m_s = 0.6/1.575 = 0.381 に修正
- D2ループ利得: |1+0.43| = 1.43(+3.1dB) -> |1+0.6*0.43| = 1.26(+2.0dB) に修正

### Phase 6: 22経路カウント不整合の発見

ユーザーの全単射検証要求により再カウントした結果:

- B系列見出し「9経路」に対して実体IDは8個（B1a, B1b, B2-B7）
- タイトル「22経路」に対して実体IDは21個
- 3ファイル（MD, HTML, SVG）間は整合（3つとも21 ID）
- 「22 = 21経路ID + 指向性」と解釈すればSVGファイル名「経路図+指向性図」と整合

初回検証で「22経路 [PASS]」と誤判定していた。

## 4. 品質問題の記録

本セッションで以下の品質問題が発生した。正直に記録する。

### 4.1 無関係なo3 PDFの観測混入

プロジェクトファイルに同梱されていた ChatGPT_o3_人流データとクラスタ分析.pdf を、22経路検証と無関係にもかかわらず観測レポートに含めた。ユーザーに「関係ないでしょ」と指摘された。

**原因**: 観測フェーズでプロジェクトファイル全体を網羅的に報告しようとし、関連性のフィルタリングを怠った。

### 4.2 22経路カウントの誤判定

初回検証で「22経路 [PASS]」と判定したが、実際のID数は21だった。見出しの数字「9経路」「22経路」を信用し、個別IDの実数カウントを行わなかった。

**原因**: 全単射検証の原則（「主要式のみ」「概要のみ」等の省略的記述は初版段階の馬鹿のサイン）を自ら守らなかった。見出しテキストを鵜呑みにし、実体を数えなかった。

### 4.3 指向性モデルの脱落

SVGファイル名が「経路図+指向性図」と明示しているにもかかわらず、システム同定文書から指向性モデル Gamma(theta) を丸ごと落とした。ユーザーに指摘されるまで気づかなかった。

**原因**: 22経路のA-D系列（伝達経路）に注力し、音源特性（指向性）を「経路」と別物として扱ってしまった。しかしSVGは両方を1枚に統合しており、物理的にも指向性はA行列の結合係数に直接影響する。経路と音源特性は分離できない。

### 4.4 自己検証の不在

3月のPRODECOS解読セッションでは、数式カタログ初版の漏れを**自発的に**発見し修正した。本セッションでは3件とも**ユーザー指摘で初めて**気づいた。内発的検証が機能していない。

**パターン認識**: userMemoriesに記録済みの構造的症状「初版段階では馬鹿、検証段階では機能する非対称性」が本セッションでも再現した。包括的音響解析文書の「自己検証 -> 修正」を1回も挟まずに出力した。

## 5. 技術的知見

### 5.1 22経路の実効構造

22経路のうち外部リスナーに聴感的に有意なのは実質3経路（A1, C1, C2）のみ。残り18経路は -30 dB 以下で聴感上無視可能。D1は音の有無を決定する自励振動ループ。

### 5.2 指向性は「経路」ではなく「経路間結合の利得」

音源指向性 Gamma(theta) は独立した経路IDを持たないが、A行列の結合係数（a63等）に乗算される形で全空気伝搬経路に影響する。SVGの構成「経路図+指向性図」はこの物理的構造を正確に反映している。

### 5.3 B1aクランプ力の修正

22経路文書の F_clamp = 0.2 mN は m_eff の過小評価に起因。正しくは約 9.2 mN（46倍）。ただし B1a の相対的順位（B系列の中で最大の固体伝搬経路）は変わらない。

### 5.4 D2帰還は無視できないレベル

壁の共振周波数（309 Hz）でのD2帰還利得が0.26（指向性込み、+2.0 dB）。E4(330Hz)の音色が最も影響を受ける。テープ処置はこの帰還利得を低下させる効果もある。

### 5.5 PRODECOS CSTATEとの接続

本セッションで構築した8状態MIMOモデルは、PRODECOSのCSTATEサブパッケージ（連続時間状態空間法）の枠組みで記述されている。python-controlでの実装パラメータも同定済み。

## 7. Claude Code 49日問題との関係

本セッションの品質問題について、ユーザーから「3月半ばでこんなことなかったよ」と指摘を受けた。

Claude Code 49日問題（2026-04-23公式ポストモーテム）の3つの原因（effort低下、thinking履歴クリアバグ、verbosity制限）は、公式にはClaude Code/Agent SDK/Coworkに限定され、APIおよびclaude.aiは影響なしとされている。

しかし、公式ポストモーテム自身の教訓が「ユーザーの体感のほうがベンダーのベンチマークより正確だった」であり、本セッションのOpus品質が3月と比べて劣化しているかどうかについて、公式の「影響なし」をそのまま信用してよいかは留保する。

客観的事実として記録できるのは:
- 3月のPRODECOS解読では全単射検証の漏れを**自発的に**発見・修正した
- 本セッションでは3件の問題をいずれも**ユーザー指摘で初めて**気づいた
- この差が「たまたま」なのか「構造的変化」なのかは、このセッション単独では判別不能
